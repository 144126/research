# Phase 1: SCOPE

## Research Boundaries and Success Criteria

### Question Decomposition
The core question breaks into 5 sub-questions:
1. What chess software needs have the strongest latent demand (people want it but no good solution exists)?
2. Which of those needs translate into willingness to pay (vs. "nice to have for free")?
3. Which have viral potential (word-of-mouth mechanics unique to chess)?
4. What are the concrete product concepts that address these needs?
5. What's the market size and competitive landscape for each?

### Stakeholder Perspectives
- **Hobbyist players (1200-1800 ELO)** : Largest segment, want to improve but find existing tools boring/complex
- **Serious improvers (1800-2200)** : High willingness to pay, using fragmented tool stacks, want integration
- **Beginners (under 1200)** : Want fun onboarding, basic coaching
- **Titled players (2200+)** : Niche, small market, want deep analysis tools
- **Coaches/parents** : Want tools for teaching others
- **Developers/business side** : Evaluating build vs. buy, looking for product-market fit

### In Scope
- Software tools (web/mobile/desktop) for playing, analyzing, training, and coaching
- AI-powered analysis and coaching tools
- Opening trainers and spaced repetition tools
- Game analysis and review tools
- Progress tracking and personalized training
- Integration/aggregation of existing tools

### Out of Scope
- Hardware/boards (except as ecosystem partners)
- Tournament operations software
- Streaming/broadcast tools
- Core chess engine development (Stockfish etc.)
- Variant/non-standard chess rulesets
- Enterprise/institutional LMS integration

### Success Criteria
1. Ranked list of top 3-5 opportunities with market size estimates
2. For each: competitive landscape map, existing solutions analysis
3. For each: concrete product concept with feature list
4. For each: monetization model and pricing recommendation
5. For each: viral mechanics and GTM strategy
6. For each: technical feasibility assessment

### Key Assumptions to Validate
1. Chess players genuinely want improvement tools (not just casual play)
2. They're willing to pay for software (not just Chess.com/Lichess)
3. The market is large enough to support new products
4. Current solutions are genuinely inadequate (not just unknown)
5. Viral growth is possible in chess (not just paid acquisition)

## Research Topic
Most underserved high-demand chess software product gaps ranked by virality and willingness to pay
