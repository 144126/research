# Phase 1: SCOPE - Research Framing

## Research Question
Which LLM API provider delivers the fastest tokens per second (tok/s) for inference as of mid-2026, across all model size tiers, and what are the architectural reasons, cost tradeoffs, and practical implications?

## Core Components
1. Speed tiers by provider and model size
2. Architectural reasons for speed differences
3. Cost-speed tradeoffs
4. Practical implications for different use cases

## Stakeholder Perspectives
- Engineers evaluating APIs for production
- Engineering managers making build-vs-buy decisions
- Technical founders selecting tech stack
- AI/ML architects designing multi-provider systems

## In Scope
All major LLM API providers, tok/s benchmarks, TTFT, architectural analysis, cost-per-token, real-world implications

## Out of Scope
Model quality benchmarks, self-hosted inference, training performance, regional availability

## Success Criteria
- 270+ sources
- 3+ sources per major claim
- Specific tok/s numbers for each provider
- Clear speed hierarchy with architectural explanations
- Actionable recommendations

## Key Assumptions
1. Public benchmarks are directionally accurate
2. Provider-reported numbers may be optimistic
3. Real-world performance varies with concurrency
4. Speed hierarchy is relatively stable in 2025-2026
